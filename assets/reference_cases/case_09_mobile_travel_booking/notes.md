# Case 09 - Mobile Travel/Booking Screens

## Source

Figma mobile wireframe reference, Travel page (`39:1117`).

## Useful Pattern

Travel screens combine discovery media, search criteria, result lists, booking forms, passenger/contact inputs, filters, and itinerary summaries.

## Repair Intent

Keep search fields and date/location pairs aligned, preserve result card scanability, and protect bottom booking CTAs. Booking forms should read as grouped decisions rather than isolated fields.

## Common Failures

- Origin/destination/date fields drift into unrelated rows.
- Filter chips wrap without consistent rhythm.
- Destination media overwhelms search or booking fields.
- Booking summary and CTA compete visually.
- Result cards lose price/time/provider alignment.
