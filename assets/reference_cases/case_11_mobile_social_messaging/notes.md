# Case 11 - Mobile Social/Messaging Screens

## Source

Figma mobile wireframe reference, Social page (`39:1120`).

## Useful Pattern

Social/messaging screens include chat lists, message threads, input bars, profiles, feeds, comments, video calls, rooms, and history lists. Each mode has different protected regions.

## Repair Intent

For chat, preserve sender/receiver alignment, bubble grouping, and sticky input bars. For social profiles and feeds, preserve media dominance without breaking action clusters or comment/list rhythm.

## Common Failures

- Message bubbles cross the sender/receiver alignment axis.
- Input bar collides with the home indicator.
- Chat list rows lose avatar/text/time alignment.
- Video call participant grid and bottom controls overlap.
- Comments and reply CTA compete for bottom space.
