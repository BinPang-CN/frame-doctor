# Case 05 - Mobile Auth/Form Screens

## Source

Figma mobile wireframe reference, Sign In/Sign Up page (`39:1111`).

## Useful Pattern

Auth screens are vertical form systems, not generic mobile cards. Protect:

- title/subtitle block
- stacked form fields
- password recovery or legal helper text
- primary CTA
- social login options
- bottom secondary account switch
- modal/bottom-sheet variants

## Repair Intent

Keep form fields equal width, equal height, and rhythmically stacked. Preserve CTA priority and keep all tappable actions clear of the home indicator. Bottom sheets should preserve handle, title, fields, and CTA order.

## Common Failures

- Form labels and inputs drift out of alignment.
- Social login buttons detach from the auth action stack.
- Primary and secondary actions have unclear weight.
- Bottom account-switch text collides with the home indicator.
- Bottom sheet content is vertically compressed.
