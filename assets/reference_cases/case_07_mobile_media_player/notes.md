# Case 07 - Mobile Media Player Screens

## Source

Figma mobile wireframe reference, Media page (`39:1114`).

## Useful Pattern

Media screens depend on artwork dominance, metadata grouping, timeline/progress alignment, and control symmetry. Player controls are functional controls, not decorative icons.

## Repair Intent

Keep artwork, title/artist metadata, progress line, playback controls, queue/lyrics/detail areas, and tab bar as separate regions. Center primary playback controls and preserve equal spacing between secondary controls.

## Common Failures

- Artwork squeezes metadata or controls into unsafe areas.
- Progress handle detaches from the track.
- Playback icons are not centered as a control cluster.
- Lyrics/detail text overlaps sticky controls.
- Media cards in lists lose consistent image/title pairing.
