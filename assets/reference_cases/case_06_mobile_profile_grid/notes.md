# Case 06 - Mobile Profile/Grid Screens

## Source

Figma mobile wireframe reference, Profile page (`39:1112`).

## Useful Pattern

Profile screens combine identity, social proof, media grids, and action buttons. Treat avatar, name, stats, tabs, gallery grid, and follow/message actions as separate but connected regions.

## Repair Intent

Preserve the profile header hierarchy first, then normalize the stats row and gallery/list grid. Action buttons should stay near the profile identity or content context they affect.

## Common Failures

- Avatar, name, and handle separate visually.
- Stats row spacing drifts and breaks comparison.
- Gallery tiles lose equal gutters or crop consistency.
- Floating follow/message controls overlap media.
- Long bio/about text compresses later content.
